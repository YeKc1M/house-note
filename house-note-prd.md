# House Note — 租房看房记录工具 / Product Requirements Document

## 1. Background & Motivation

### Problem Statement
在成都租房看房过程中，需要同时对比多个小区、多套房源。看房维度复杂（通勤、朝向、户型、家具家电、噪音等），用备忘录或 Excel 记录既慢又容易遗漏重点，且不同时间、不同房源的格式不统一，事后对比困难。

### Target Users
- **主要用户：自己。** 工具以单人使用为核心，先解决个人看房时的信息结构化记录与回溯问题。

### Why Now
目前看房依赖零散笔记，信息分散、格式混乱，需要一款支持「自定义维度 + 层级分组 + 移动端随时记录 + 电脑端偶尔查看修改」的工具。

---

## 2. Scope

### In Scope
- Android 端：核心记录入口，支持模板编辑、实例创建、数据录入、分组浏览、导入导出。
- Windows 端：浏览器可访问的网页版或轻量 EXE，用于查看和修改数据，支持局域网同步。
- 自定义维度模板系统（支持无限嵌套层级）。
- 自定义分组规则（例如：小区 → 房子）。
- 缩略图卡片自定义显示字段。
- 实例可偏离模板（仅影响当前实例，不修改模板）。
- 数据导入/导出：YAML 格式，支持微信文件分享导入导出。
- 局域网点对点同步（同一 WiFi 下，电脑无客户端时输 IP 直连；有客户端时自动配对）。
- 离线可用，同步在有网络时手动触发。
- 冲突处理：检测到手机和电脑同一数据冲突时，警告并让用户选择以哪一侧为准覆盖。

### Out of Scope
- 拍照关联看房维度（P2，暂时不做）。
- 房源对比视图（P2，暂时不做）。
- 多用户协作、权限管理。（不做）
- 云端自动同步（如服务器备份）。
- 地图集成、价格走势分析。
- 性能极限优化与自动数据迁移（当前阶段数据量小，未来再议）。

## 3. Core Features & User Flows

### Feature 1: 自定义维度模板
- **User Story:** 作为看房者，我想要自定义看房时关注的维度模板，以便在不同看房场景中按需记录信息。
- **Key Steps / Interactions:**
  1. 用户进入「模板管理」，新建或编辑模板。
  2. 模板由「维度项」组成，每个维度项包括：名称、类型（文本 / 单选 / 布尔 / 数字 / 子维度组等）。
  3. 维度项支持无限嵌套（例如：小区 → 通勤 → 地铁 → 几号线）。
  4. 模板保存后，在看房实例化时直接调用。

### Feature 2: 按模板创建看房实例
- **User Story:** 作为看房者，我想在看房时根据模板快速创建实际的层级实例，以便边看房边打分记录。
- **Key Steps / Interactions:**
  1. 用户在列表页选择模板，新建一个顶层实例（例如小区），填入该层级下的通用信息。
  2. 在该实例下，若模板定义了下一级（例如房子），用户可以继续新建下一级实例并录入数据。
  3. 若模板继续定义了更深层级（例如卧室），用户可以继续在对应实例下新建更深层的实例并录入数据。
  4. 用户在记录过程中，可以动态给当前实例添加「模板外字段」，仅影响该实例。

### Feature 3: 自定义分组与缩略图展示
- **User Story:** 作为看房者，我想按自定义层级分组浏览实例，并在列表卡片上快速看到最关心的字段，以便高效对比和筛选。
- **Key Steps / Interactions:**
  1. 用户可配置分组规则：指定模板中哪些层级作为分组节点（例如把某一级设为 group 后，列表先展示该层级的所有实例卡片，点击进入再展示其下的子实例）。
  2. 用户可自定义各个层级卡片的缩略图显示字段（例如顶层卡片显示位置和地铁线，下一级卡片显示楼层和朝向）。
  3. 列表支持层级跳转与返回。

### Feature 4: YAML 导入导出（含微信分享）
- **User Story:** 作为看房者，我想要把数据导出成可读的文件，并通过微信发给电脑或备份，以便跨设备查看和修改。
- **Key Steps / Interactions:**
  1. 用户在 Android 端点击「导出」，生成 YAML 文件。
  2. 通过微信「发送到文件传输助手」或邮件等方式传到电脑。
  3. 在 Windows 端打开 App，导入 YAML，查看和修改数据。
  4. 修改后的 YAML 再通过微信传回手机，在 Android 端导入覆盖。
  5. 导入时若检测到字段级冲突（同一实例的 same 字段在手机和 YAML 中都被修改过），弹出警告，由用户选择「保留手机版本」或「保留导入版本」。
  6. 实例级别的模板外字段必须随 YAML 一起导出/导入。

### Feature 5: 局域网点对点同步
- **User Story:** 作为看房者，在同一 WiFi 下，我想要手机和电脑快速同步数据，而不是每次都走微信传文件。
- **Key Steps / Interactions:**
  1. Android 端打开「局域网同步」，显示当前设备 IP 或二维码。
  2. **场景 A（电脑无客户端）**：电脑浏览器输入 Android 设备 IP 地址，直连到 Web 界面，进行数据查看和修改，修改后自动/手动同步回手机。
  3. **场景 B（电脑有客户端）**：Windows 端打开客户端，同一网络下自动发现 Android 设备，一键配对同步。
  4. 同步触发为手动，用户确认后再执行。
  5. 冲突处理与 Feature 4 一致：检测到冲突时弹窗让用户选择覆盖方向。

## 4. Non-Functional Requirements

### Performance
- 当前目标数据量：十几个小区、几十套房。
- 列表加载、模板展开、实例创建应在 1 秒内响应，无需考虑超大规模优化。

### Security / Privacy
- 纯本地工具，无用户账号体系。
- 数据不强制加密，但 YAML 文件作为个人数据应妥善保管。

### Scalability
- 现阶段单机本地运行即可。未来数据量增长后，再评估数据迁移或分页加载方案。

### Accessibility / Compliance
- 以自用为主，基础可读性即可，无需严格的 accessibility 标准。

### Constraints
- **平台约束**：Android 端优先完成；Windows 端为浏览器网页版或轻量 EXE。
- **网络约束**：核心功能必须完全离线可用；同步仅在 WiFi/局域网或手动导入导出时进行。
- **存储格式约束**：底层数据持久化与导入导出统一使用 YAML，兼顾人类可读性与结构化。
- **技术栈**：暂不做硬性限制，以实现功能为首要目标。

## 5. Prioritization

### P0 — Must-have for Launch
- A. 自定义维度模板（支持无限嵌套）
- B. 按自定义规则分组浏览（如小区 → 房子）
- C. 缩略图卡片自定义显示字段
- D. 局域网 Android ↔ Windows 同步
- E. YAML 导入导出（含微信文件分享路径）

### P1 — Should-have (Fast-follow)
- 模板编辑在 Windows 端的友好 UI（目前手机可编辑，电脑至少能查看和改实例）
- 更优雅的冲突合并界面（当前阶段为简单覆盖选择）

### P2 — Nice-to-have / Future
- 数据量大了之后的性能优化与迁移

## 6. Acceptance Criteria

- [ ] 用户可以在 Android 端创建、保存、编辑一个包含多层嵌套的维度模板。
- [ ] 用户可以在列表中按模板新建任意层级的实例，并在允许存在子层级的实例下继续新建下一级实例。
- [ ] 用户在录入任意层级实例时，可以给该实例添加模板之外的字段，且该字段仅影响当前实例。
- [ ] 用户可以配置分组规则，列表按分组层级展示（如先展示小区卡片，点击进入展示房子卡片）。
- [ ] 用户可以自定义每个层级卡片上显示的缩略图字段。
- [ ] 用户可以将全部数据导出为 YAML 文件，并通过微信发送到电脑。
- [ ] 电脑修改后的 YAML 传回手机导入后，数据正确覆盖；若存在字段冲突，App 弹窗提示并让用户选择以哪一侧为准。
- [ ] 实例级别的模板外字段在导出/导入后保留完整。
- [ ] 在相同 WiFi 下，Android 与 Windows 能通过 IP 直连或客户端自动配对完成手动同步。
- [ ] 所有核心记录功能在完全离线状态下正常运行。
